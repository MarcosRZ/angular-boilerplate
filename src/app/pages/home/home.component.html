Home This component is a Page

After dive into Angular, I was working with NextJS / React. I really liked the logical structure proposed by Zeit team in their 'NextJS' project. I think that it is clear and intuitive. Of course, there is no restriction or dependency on this. Feel free to modify it to fit your needs. And if you think that it can be improved in any way, please, let me know.

In the 'app' directory of this project you will see two main directories:

Pages

Angular components that have the general main purpose of showing a mainpage.

Examples: Home, About, Contact...

Components

Angular components whose role is to implement a small part of the UI/UX. I.E.:

Examples: SearchBox, PhotoCanvas, PostsList...

These two directories are the most important. But as you could find out, there are some others:

Core

Angular components that are traversal to the whole front. Mainly, static components.

Examples: Header, Navbar, Footer...
